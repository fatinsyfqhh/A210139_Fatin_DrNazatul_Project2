
### Project Description
Project 2 represents the final production-ready mobile application solution for this course. This project builds directly upon earlier milestones by blending the structured local storage data framework of **Lab 5 (Room Database)** with remote web networking, hardware-level tracking systems, and active cloud data streaming pipelines into a highly polished, production-ready product.

### Core Implementation Modules
* **Hybrid Storage Architecture (Room):** Full offline-first capabilities using localized Room DB schemas for seamless internal data management.
* **Remote Networking & APIs:** Networking layer (Retrofit/Ktor) to run asynchronous external HTTP queries, serialize JSON formats, and parse live remote assets.
* **Hardware Sensor Integration:** Native interaction with on-device hardware components and sensors to track real-world environment context natively.
* **Cloud Storage Synchronizations:** Integration with remote cloud backends (e.g., Firebase or online REST services) for global data persistence and real-time syncing.
* **Advanced Background Engineering:** Managed threading routines using Kotlin Coroutines and Flows to guarantee asynchronous processing without causing UI stutters.

### Built With
* Full-Stack Kotlin & Jetpack Compose Declarative Toolkits
* Room Database Architecture & Core Android Sensor Frameworks
* Retrofit Client / REST API Networking Pipelines
* Remote Cloud Systems & Asynchronous Flow Architectures
* Android Studio

### Setup & Deployment
1. Clone this repository locally.
2. Open the complete codebase inside **Android Studio**.
3. Sync Gradle and verify any required external service credentials (API keys, cloud json assets, etc.).
4. Run the production release build on an active device or emulator.
